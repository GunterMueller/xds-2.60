- редактор проекта, наряду со старым синтаксисом
  +option
  -option
  #equ = value
  #equ ! value
  :option+
  :option-
  понимает новый:
  -option+
  -option-
  -option:+
  -option:-
  -equ = value
  -equ ! value
  При изменении опции он новое значение пишет в новом синтаксисе.
- в файле Options.cfg можно написать
  +:option
  -:option
  +:~option (именно в таком порядке и без пробела между +,: и ~)
  #:equation...
  $:equation...
  $:equation...
  и это будет значить, что при вставлении нового значения в проектный
  файл будет сгенерировано -name:+ или -name!val (если значение уже было,
  наличие : берется из старого значения, а не из options.cfg)
- чуть изменена отрисовка текущей строки в редакторе опций
- сделано, чтобы если проект открыт в редакторе, то, если его изменить

  бы на месте
